📚 What Each Part Means
State

📅 Conversation Phase

A State represents a major phase or stage of the conversation. The agent moves through states sequentially as the conversation progresses.

id Unique identifier for this state
label Human-readable name shown in UI
execution_mode "sequential" or "flexible" task order
status Current progress of this phase
pending active completed skipped
Task

☑ Action to Perform

A Task is a specific action the agent must complete within a state. Each task has clear instructions and may collect multiple pieces of information.

id Unique identifier for this task
label Short description of what to do
instruction Detailed guidance for the agent
required Must complete to proceed?
pending in_progress completed skipped
Deliverable

📦 Data to Collect

A Deliverable is a specific piece of information the agent extracts from the conversation. Once collected, it's stored with a confidence score.

key Variable name for the collected data
type string, number, boolean, or enum
value The collected value (null until filled)
confidence How certain the agent is (0.0 - 1.0)
pending partial completed skipped
💬 How It Works in a Real Conversation
1

Agent Loads Plan

When the conversation starts, the agent loads the plan template and activates the first state.

2

Agent Works on Tasks

The agent follows task instructions and guides the conversation naturally toward collecting deliverables.

3

Deliverables Are Collected

As the user speaks, the agent extracts information and fills in deliverable values with confidence scores.

4

Progress Advances

When all required deliverables in a task are complete, the task is marked done. When all tasks finish, the next state activates.

5

UI Updates in Real-Time

The frontend receives live updates showing current state, active task, and collected values - providing transparency to the user.
